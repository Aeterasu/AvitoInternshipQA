## Неконсистентное наименование JSON полей

### Приоритет: Low

- В POST `https://qa-internship.avito.com/api/1/item` используется `sellerID`
- Во всех остальных запросах используется `sellerId` (с маленькой буквой d)
- Это может привести к ошибкам, если при чтении JSON-файла не указать `PropertyNameCaseInsensitive = true` чтобы сериализатор игнорировал разницу между верхним/нижним регистром

## GET запрос для одного ресурса возвращает массив вместо объекта

### Приоритет: Medium

### Эндопинты:
* `GET /api/1/item/{id}`
* `GET /api/1/statistic/{id}`

Эндпоинты для получения одного объекта по уникальному ID возвращают массив содержащий один объект - вместо просто одного объекта. Для GET запроса с уникальным ID ожидается возврат единичного JSON объекта (соответствующего ID), однако возвращается JSON массив.

**Ожидаемый результат:**
- Возвращается единичный JSON объект `{...}`

**Фактический результат:**
- Возвращается JSON массив `[...]`, содержащий один или более элементов

- Код, ожидающий единичный JSON объект, может выдать исключение
- Из-за этого при работе с API, приходится писать код, обрабатывающий массив вместо прямого доступа к объекту
