## Тестовые кейсы

### Эндпоинт 1 - POST - Сохранить объявление

**Эндпоинт:** https://qa-internship.avito.com/api/1/item

**Метод:** `POST`

**Описание:** Сохранить новое объявление и отправить его в бэкенд

* **Кейс 1 - Правильный ввод**
    * **Ожидаемый статус:** `200 OK`
    * **Ожидаемое поведение:** Создано новое объявление, статус возвращает сгенерированный `id`.

* **Кейс 2 - Некорректный JSON**
    * **Ожидаемый статус:** `400 Bad Request`
    * **Ожидаемое поведение:** Ответ выдает ошибку, некорректный формат JSON-файла.

---

### Эндпоинт 2 - GET - Получить объявление по ID

**Эндпоинт:** https://qa-internship.avito.com/api/1/item/{id}

**Метод:** `GET`

**Описание:** Возвращает существующее объявление с правильным id

* **Кейс 1 - Правильный ввод**
    * **Ожидаемый статус:** `200 OK`
    * **Ожидаемое поведение:** Возвращает корректный JSON-объект с объявлением.

* **Кейс 2 - Некорректный ID**
    * **Ожидаемый статус:** `400 Bad Request`
    * **Ожидаемое поведение:** Ответ выдает ошибку - id не соответствует формату ID.

* **Кейс 3 - Несуществующий ID**
    * **Ожидаемый статус:** `404 Not Found`
    * **Ожидаемое поведение:** Ответ выдает ошибку - id соответствует формату, но такого объявление не существует.

---

### Эндпоинт 3 - GET - Получить все объявления пользователя

**Эндпоинт:** https://qa-internship.avito.com/api/1/{sellerID}/item

**Метод:** `GET`

**Описание:** Возвращает все объявления пользователя с указанным ID

* **Кейс 1 - Правильный ввод**
    * **Ожидаемый статус:** `200 OK`
    * **Ожидаемое поведение:** Возвращает корректный JSON-объект - массив со всеми объявлениями данного пользователя.

* **Кейс 2 - Некорректный ID**
    * **Ожидаемый статус:** `400 Bad Request`
    * **Ожидаемое поведение:** Ответ выдает ошибку - id некорректен, или такого пользователя не существует.

---

### Эндпоинт 4 - GET - Получить статистику по объявлению

**Эндпоинт:** https://qa-internship.avito.com/api/1/statistic/{id}

**Метод:** `GET`

**Описание:** Возвращает объект со статистикой объявление с указанным ID

* **Кейс 1 - Правильный ввод**
    * **Ожидаемый статус:** `200 OK`
    * **Ожидаемое поведение:** Возвращает корректный JSON-объект - статистику существующего объявления.

* **Кейс 2 - Некорректный ID**
    * **Ожидаемый статус:** `400 Bad Request`
    * **Ожидаемое поведение:** Ответ выдает ошибку - id некорректен, или такого объявления не существует.

* **Кейс 3 - Несуществующий ID**
    * **Ожидаемый статус:** `404 Not Found`
    * **Ожидаемое поведение:** Ответ выдает ошибку - id корректного формата, но такого объявления не существует.
